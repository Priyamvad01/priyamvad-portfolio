# SEO

The project uses the Next.js Metadata API and generated assets to create a deployment-ready SEO foundation.

## Metadata System

SEO metadata is centralized through `lib/seo.ts`.

Each major page defines unique metadata:

- Home
- About
- Skills
- Projects
- Healthcare AI System Case Study
- Certifications
- Terminal
- Contact

Metadata includes:

- Title
- Description
- Keywords
- Canonical URL
- Open Graph data
- Twitter card data
- Robots directives

## Generated SEO Routes

The application generates:

```txt
/robots.txt
/sitemap.xml
/manifest.webmanifest
/opengraph-image
/twitter-image
/icon
/apple-icon
```

These are generated by Next.js route files under `app/`.

## Structured Data

## Canonical URL

The canonical base URL comes from:

```txt
NEXT_PUBLIC_SITE_URL
```

Set this value in production so canonical links, sitemap URLs, and social metadata point to the deployed domain.

## SEO Maintenance

When adding a new page:

1. Add route-level metadata with `createMetadata`.
2. Add the route to `app/sitemap.ts` when needed.
3. Use a descriptive title and realistic technical description.
4. Keep page headings semantic and recruiter-readable.
